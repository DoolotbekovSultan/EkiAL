# Response Models

## Назначение
Папка `response_models` содержит стандартизированные модели для работы с ответами API. Обеспечивает единообразную структуру обработки данных от сервера.

## Структура
response_models/
├── base_response.dart
├── paginated_response.dart
├── error_response.dart
└── success_response.dart

## Описание файлов

### base_response.dart
Базовая модель ответа API. Содержит:
- Стандартные поля всех API ответов
- Обработку статусов успеха/ошибки
- Методы для проверки валидности данных

### paginated_response.dart
Модель для пагинированных данных. Содержит:
- Структуру пагинации (страницы, лимиты)
- Методы для навигации по страницам
- Валидацию пагинационных параметров

### error_response.dart
Модель для обработки ошибок API. Содержит:
- Стандартизированную структуру ошибок
- Коды и сообщения об ошибках
- Методы для преобразования в доменные Failure

### success_response.dart
Модель для успешных ответов API. Содержит:
- Обработку данных успешных операций
- Мета-информацию о запросе
- Методы для извлечения полезных данных

## Преимущества

- **Стандартизация** - единая структура всех API ответов
- **Type-safe** - строгая типизация данных ответов
- **Валидация** - проверка целостности данных от сервера
- **Упрощение** - легкая работа с пагинацией и ошибками
- **Тестируемость** - простые моки для тестирования

## Best Practices

1. Используйте generic типы для переиспользуемости
2. Валидируйте обязательные поля при создании моделей
3. Реализуйте методы для удобного доступа к данным
4. Обрабатывайте edge cases (пустые списки, null значения)
5. Тестируйте все возможные сценарии ответов
