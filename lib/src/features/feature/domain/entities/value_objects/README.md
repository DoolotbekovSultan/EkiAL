/**
 * Value Objects (Объекты-Значения)
 * 
 * Этот файл содержит объекты-значения (Value Objects) - неизменяемые объекты,
 * которые идентифицируются по своим атрибутам, а не по идентификатору.
 * Используются для инкапсуляции валидации и бизнес-логики примитивных типов.
 * 
 * ## Основные обязанности:
 * - Инкапсуляция бизнес-правил для примитивных значений
 * - Валидация данных при создании объекта
 * - Обеспечение иммутабельности и типобезопасности
 * - Реализация семантического равенства по значению
 * - Предоставление доменно-специфичного поведения
 * 
 * ## Ключевые компоненты:
 * 
 * ### ValueObject
 * Базовый класс для всех объектов-значений:
 * - `value` - основное значение объекта
 * - `validate()` - валидация значения при создании
 * - Семантическое равенство по значению, а не по ссылке
 * 
 * ### Специализированные Value Objects:
 * - **Email** - валидация email адресов
 * - **Password** - валидация и хэширование паролей
 * - **Money** - работа с денежными суммами и валютами
 * - **DateRange** - работа с диапазонами дат
 * - **Coordinate** - географические координаты
 * 
 * ## Особенности:
 * - Immutable (неизменяемые) объекты
 * - Валидация при создании (fail-fast подход)
 * - Семантическое равенство (equals/hashCode по значению)
 * - Доменно-специфичное поведение и валидации
 * - Легкость тестирования бизнес-правил
 * 
 * ## Best Practices:
 * - Делайте все Value Objects неизменяемыми
 * - Валидируйте данные в конструкторе
 * - Реализуйте правильные equals и hashCode
 * - Используйте для примитивов с бизнес-логикой
 * - Документируйте бизнес-правила валидации
 * - Тестируйте граничные случаи и ошибки
 * - Используйте композицию для сложных Value Objects
 */s
